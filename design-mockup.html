What changed — and why it isn't a library swap

Every pixel on the right uses your existing CSS variables (teal accent, surfaces, text levels, radii, shadows, Inter + JetBrains Mono). These are visual-design moves, applied in the CSS you already own.

1

Type hierarchy

Bigger, tighter KPI numbers (800 weight, −.035em) and a clear label/value/footer rhythm so the eye lands on the metric first.

2

Intentional accent

One "feature" KPI in teal, tinted icon chips, an active-nav rail. The brand colour leads instead of sitting only on buttons.

3

Depth & motion

Layered shadows, hover lift on cards and rows, a real app-shell elevation — using your `--shadow-*` and `--motion-*` tokens.

4

Richer rows

Customer avatars + phone, zebra-free hover, right-aligned mono money, softer header band. Same data, more scannable.

5

Density & spacing

Consistent 13–16px gaps and 16px radii give breathing room without losing the compact, ops-grade feel.

6

So where's shadcn?

The middle column. Notice it's clean but monochrome and brand-less by default — near-black buttons, neutral badges. It would need the same visual work to feel like "Bika", so the library isn't what removes "generic".
